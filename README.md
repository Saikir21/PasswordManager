# Local Password Manager (Python / SQLite / Fernet)

Локальный менеджер паролей с консольным интерфейсом (CLI), построенный на принципах безопасного хранения данных и разделения архитектурных слоев.

## 🚀 Быстрый старт и логика работы

### 1. Автоматическая инициализация (Первый запуск)
Вам не нужно вручную создавать базы данных или таблицы. При первом запуске скрипта:
1. Система автоматически создаст локальную базу данных `passwords.db`.
2. Создадутся таблицы для авторизации (`auth`) и хранения аккаунтов (`accounts`).
3. Если хранилище новое, система предложит создать ваш уникальный Мастер-пароль.

### 2. Критически важно: Настройка безопасности (SECRET_PEPPER.env)
В репозитории **намеренно отсутствует** файл конфигурации `SECRET_PEPPER.env`. Это сделано в целях безопасности, чтобы ваш секретный "перец" не улетел в публичный доступ (файл внесен в `.gitignore`).

Если запустить проект без настройки, система выдаст контролируемую ошибку: `ValueError: SECRET_PEPPER не найден!`.

**Что нужно сделать перед запуском:**
1. В корневой папке проекта создайте файл с именем `SECRET_PEPPER.env`.
2. Добавьте в него одну строчку со своей секретной ключевой фразой:
   
   ```env
   SECRET_PEPPER=ваша_секретная_случайная_строка

# Архитектура проекта

main.py — Управляющий консольный интерфейс (CLI), обработка выбора пользователя (1-6) и логика сессии.

core.py — Криптографическое ядро (Fernet, PBKDF2) и прямые SQL-запросы к базе данных.

models.py — Описание сущностей (Data Classes) для безопасной передачи данных между слоями.

config.py — Централизованные настройки приложения (таймаут буфера, итерации KDF, пути к файлам).

# Функции безопасности

Симметричное шифрование: Пароли сервисов шифруются алгоритмом AES в режиме CBC (библиотека cryptography.fernet).

Усиление ключа (KDF): Мастер-пароль превращается в ключ шифрования с помощью PBKDF2HMAC (SHA-512, 600 000 итераций).

Разделение секретов (Соль и Перец): Хэш мастер-пароля защищен уникальной солью (хранится в БД) и общим перцем (хранится строго локально в .env).

Защита от тайминг-атак: Проверка мастер-пароля идет через hmac.compare_digest с искусственной задержкой времени при ошибке.

Параметризованные SQL-запросы: Полная защита от SQL-инъекций благодаря плейсхолдерам (WHERE service = ?).

Безопасность буфера: Автоматический сброс скопированного пароля из буфера обмена через 20 секунд.
